

## Создать репозиторий Через веб‑интерфейс GitHub

**Шаг 1.** Войдите в свой аккаунт 

**Шаг 2.** Создайте новый репозиторий:

- кликните значок **«+»** в правом верхнем углу экрана;
- выберите **New repository**.

**Шаг 3.** Заполните параметры:

- **Repository name** — уникальное имя репозитория (латиница, цифры, дефисы, подчёркивания; без пробелов);
- **Description** (опционально) — краткое описание проекта;
- **Privacy** — выберите:
  - **Public** (публичный: виден всем, бесплатный);
  - **Private** (приватный: виден только вам и указанным участникам, может требовать платной подписки);
- **Initialize this repository with:**
  - отметьте **Add a README file**, чтобы создать файл `README.md` с описанием проекта;
  - выберите **.gitignore**, если нужно исключить временные файлы (например, для Python, Java и т. д.);
  - укажите **License** (например, MIT, Apache 2.0), чтобы защитить права на код.

**Шаг 4.** Нажмите **Create repository**.

Теперь у вас есть удалённый репозиторий на GitHub.

###  Удаление репозитория на GitHub (через веб‑интерфейс)

**Важно:**

- удалить можно только репозитории, которыми вы владеете (у вас есть права администратора);
- удаление **необратимо** — исчезают код, коммиты, issues, пул‑реквесты, вики и т. д.;
- локальная копия на вашем компьютере останется (если она есть), но связь с удалённым репозиторием прервётся.

**Шаги:**

1. Войдите в свой аккаунт на 

.

Перейдите в список репозиториев: кликните на иконку профиля в правом верхнем углу → выберите *Repositories*.

Найдите нужный репозиторий и откройте его.

Перейдите в настройки: нажмите вкладку *Settings* (справа в верхнем меню).

Прокрутите страницу до раздела *Danger Zone* (внизу).

Нажмите кнопку *Delete this repository*.

Подтвердите удаление:

- прочитайте предупреждения;
- введите полное имя репозитория (например, `ваш_логин/название_репозитория`);
- нажмите *I understand the consequences, delete this repository*.

После подтверждения репозиторий будет удалён. Вы увидите уведомление об успехе.
